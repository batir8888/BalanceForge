# BalanceForge â€” Code Generation Guide

This guide explains how to generate strongly-typed runtime data assets from your Balance Tables and how to use them in game code.

## Why code generation?

A live `BalanceTable` asset stores each cell value as `object` inside a `Dictionary<string, object>`. This means:
- Every `int`, `float`, `bool` is **boxed** onto the heap
- Reading a value requires a dictionary lookup + type cast
- Unity serialises the table as nested string dictionaries

The generated asset uses a **Struct of Arrays (SoA)** layout instead. Each column becomes one plain typed array on the `ScriptableObject`:

```
// Design-time (editor only)          // Runtime (generated, zero boxing)
BalanceTable                          CharacterStatsData
  List<BalanceRow>                      int[]    Hp
    {hp: (object)100, speed: ...}       float[]  Speed
    {hp: (object)80,  speed: ...}       string[] Name
    ...                                 ...
```

Accessing `data.Hp[i]` is a direct array read â€” no dictionary, no cast, no allocation.

## Step 1 â€” Generate code from the editor

1. **Window â†’ BalanceForge â†’ Table Editor**
2. Load your `BalanceTable` asset via the top-left object field
3. Click **`<> Generate`** in the toolbar
4. Choose an output folder (default: `Assets/BalanceForge/Generated/`)
5. Click **Generate + Bake**

Unity compiles the new `.cs` file. After compilation a `.asset` file with your baked data is created automatically in the same folder.

**Files produced for a table named `CharacterStats`:**

| File | Contents |
|---|---|
| `CharacterStatsData.cs` | Generated `ScriptableObject` class |
| `CharacterStatsData.asset` | Baked data ready to reference at runtime |

## Step 2 â€” What the generated class looks like

For a table with columns `Name (String)`, `Hp (Integer)`, `Speed (Float)`, `IsActive (Boolean)`:

```csharp
// AUTO-GENERATED by BalanceForge â€” do not edit manually.
// Layout: Struct of Arrays â€” one typed array per column.
namespace BalanceForge.Generated
{
    public class CharacterStatsData : ScriptableObject
    {
        [SerializeField] public string[] Name     = System.Array.Empty<string>();
        [SerializeField] public int[]    Hp       = System.Array.Empty<int>();
        [SerializeField] public float[]  Speed    = System.Array.Empty<float>();
        [SerializeField] public bool[]   IsActive = System.Array.Empty<bool>();

        /// <summary>Number of rows baked into this asset.</summary>
        public int Count => Name?.Length ?? 0;

        /// <summary>
        /// Returns the first row index for which predicate returns true, or -1.
        /// </summary>
        public int FindIndex(Predicate<int> predicate) { ... }
    }
}
```

Column names are converted to **PascalCase** identifiers:

| Column ID | C# field |
|---|---|
| `hp` | `Hp` |
| `hp_max` | `HpMax` |
| `is-active` | `IsActive` |
| `3d_speed` | `_3dSpeed` |

## Step 3 â€” Reference the asset in your MonoBehaviour

Drag the generated `.asset` onto a `[SerializeField]` of the matching type:

```csharp
using BalanceForge.Generated;
using UnityEngine;

public class CharacterManager : MonoBehaviour
{
    [SerializeField] private CharacterStatsData stats;

    void Start()
    {
        Debug.Log($"Loaded {stats.Count} characters.");
    }
}
```

## Common usage patterns

### Read a value by index

```csharp
int hp    = stats.Hp[i];
float spd = stats.Speed[i];
string nm = stats.Name[i];
```

All reads are direct array accesses â€” no boxing, no dictionary lookup.

### Iterate over all rows

```csharp
for (int i = 0, n = stats.Count; i < n; i++)
{
    if (stats.IsActive[i])
        SpawnCharacter(stats.Name[i], stats.Hp[i], stats.Speed[i]);
}
```

Because all `Hp` values are contiguous in memory, this loop is cache-friendly. Compare to the old approach where each row was a separate heap object.

### Find a row by condition

```csharp
int idx = stats.FindIndex(i => stats.Name[i] == "Warrior");
if (idx >= 0)
{
    int warriorHp = stats.Hp[idx];
}
```

`FindIndex` returns the first matching row index, or `-1` if not found.

### LINQ over a column

```csharp
using System.Linq;

int maxHp  = stats.Hp.Max();
float avgSpeed = stats.Speed.Average();
int activeCount = stats.IsActive.Count(b => b);
```

Because each column is a plain array, all standard LINQ and `System.Array` methods work directly.

### Find all rows matching a condition

```csharp
var fastCharacters = new List<int>(); // store indices
for (int i = 0; i < stats.Count; i++)
    if (stats.Speed[i] > 5f)
        fastCharacters.Add(i);
```

### Load via Resources

Place the `.asset` inside a `Resources/` folder and load at runtime:

```csharp
var stats = Resources.Load<CharacterStatsData>("CharacterStats/CharacterStatsData");
```

### Load via Addressables (recommended)

```csharp
[SerializeField] private AssetReferenceT<CharacterStatsData> statsRef;

async void Load()
{
    CharacterStatsData stats = await statsRef.LoadAssetAsync().Task;
}
```

## Re-generating after data changes

When you edit rows in the Balance Table editor and save, the postprocessor **automatically re-generates** the `.cs` file and schedules a re-bake â€” but only for tables that have been generated at least once (opt-in).

You can also trigger it manually: click **`<> Generate`** â†’ **Generate + Bake** again. The existing `.asset` is replaced.

---

## Important constraints

- **Do not edit** the generated `.cs` file â€” it will be overwritten on the next generation.
- The generated asset is **read-only at runtime** by design (all fields are serialised arrays). To modify data at runtime, write to the `BalanceTable` in the editor and re-bake.
- The **column order** in the generated class matches the column order in the table definition at generation time. Re-ordering columns and regenerating produces a new `.cs` â€” any old `.asset` referencing the previous field layout will need to be re-baked.
